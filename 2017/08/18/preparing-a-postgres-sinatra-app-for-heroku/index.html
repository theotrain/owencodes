PREPARING A POSTGRES SINATRA APP FOR HEROKU

Steps for getting your Postgres-backed Sinatra App working on Heroku…

1 – Gemfile

Specify a ruby version and add puma

Ruby
1
2
3
4
5
source "https://rubygems.org"
ruby "2.4.0"
...
gem "puma"
...

specifying a Ruby version: https://devcenter.heroku.com/articles/ruby-versions

2 – Procfile

Create a Procfile in the main directory of your App. My file contains the following

1
web: bundle exec puma -t 5:5 -p ${PORT:-3000} -e ${RACK_ENV:-development}

Procfiles for puma server: https://devcenter.heroku.com/articles/deploying-rails-applications-with-the-puma-web-server

More on Procfile: https://devcenter.heroku.com/articles/procfile

3 – Distinguish Development and Production Code

In my app I’m using “sinatra/reloader” so i don’t have to restart the server every time there is a change to the code. We don’t need reloader in production, so in our main application file we can specify we only need reloader in development. also_reload is only provided as an example, if you are already using it.

Ruby
1
2
3
4
configure(:development) do
  require "sinatra/reloader"
  also_reload "other_page_to_reload.rb"
end

Wherever you establish your database connection using PG.connect, replace with this:

Ruby
1
2
3
4
5
@db = if Sinatra::Base.production?
        PG.connect(ENV['DATABASE_URL'])
      else
        PG.connect(dbname: "my_local_database_name")
      end

4 – Create App & Database on Heroku

Create a new app using CLI, or you can do it through Heroku website, as you like.

1
2
3
4
5
6
7
$ heroku apps:create 1979-my-garbage-app<pre>
 
Create Apps from the CLI: <a href="https://devcenter.heroku.com/articles/creating-apps">https://devcenter.heroku.com/articles/creating-apps</a>
 
Create database on Heroku
 
<pre class="nums:false">$ heroku addons:create heroku-postgresql:hobby-dev

create postgres db: https://devcenter.heroku.com/articles/heroku-postgresql#create-a-new-database

Assuming i have a local file called schema.sql i can use it to add the basic tables and schema to my new database with:

1
2
3
4
5
$ heroku pg:psql &lt; schema.sql<?pre>
 
If you want to create the tables manually, or add some data with SQL statements, you can connect to the database with psql...
 
<pre class="nums:false">$ heroku pg:psql

5 – Close Connections When Using Heroku Free plan

Heroku free plan limits the number of simultaneous database connections. To decrease the chances of exceeding that limit, add this stuff:

Add to the main application file:

Ruby
1
2
3
after do
  @storage.disconnect
end

…where @storage is an object that handles our database connection.

And add to the class that @storage refers to:

Ruby
1
2
3
def disconnect
  @db.close
end

…where @db is the Postgres connection object as we saw above in section 3

6 – Wrapping Up

– You can use heroku local to test the app locally.
– Make sure you run bundle install
– Commit any changes and deploy to heroku

1
$ git push heroku master

Good luck!
