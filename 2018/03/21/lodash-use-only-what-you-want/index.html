LODASH - USE ONLY WHAT YOU WANT

If you’re using Node.js, lodash makes it easy to use just the parts you want.

JavaScript
1
2
3
4
5
// Load method categories.
var array = require('lodash/array');
// Load single method
var at = require('lodash/at');

Easy! (more details)

But I’m Not Using Node!

If you just want to include some javascript in an HTML page, your obvious choices are either: the core build (~4kb) or the full build (~24kb)

If the functions you want are included in the core build (check here), you might want to just use that. It’s a bargain at 4kb. If what you need is NOT included in the core, you might want to do a custom build.

Making a Custom Lodash Build

First install the lodash command line tool.

1
$ npm i -g lodash-cli

Navigate to your working directory and enter something like the below, where ‘include’ is followed by a list of lodash functions you need.

1
$ lodash include=random,clamp,find

The tool will write 2 files to the working directory: ‘lodash.custom.js’ and ‘lodash.custom.min.js’. Include as normal to access your functions.

XHTML
1
<script src='lodash.custom.min.js'></script>

You can also include entire categories of lodash functions like so:

1
$ lodash category=collection,function

or a whole category plus a few miscellaneous functions like so:

1
$ lodash category=array plus=random,template

See lodash’s custom builds page for all the details.

In terms of filesize reduction, your mileage may vary. But it’s quick and painless to give it a try.
